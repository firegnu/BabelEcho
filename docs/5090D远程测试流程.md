本文记录 BabelEcho 在 5090D 上做真实 DeepSeek + 本地 TTS 验证的固定流程。目标是让任意新 session 进入项目后，都能按同一套步骤复现实验，而不是凭聊天记录回忆命令。

## 适用范围

- 本机 MacBook 负责代码修改、测试、提交和 push。
- 5090D 负责真实运行：DeepSeek adapt、CosyVoice TTS、assemble、publish。

- 真实 runtime config、DeepSeek key、TTS wrapper 配置和生成音频均保持 ignored，不提交。

## DeepSeek前置质量门禁

`normalize` 阶段会写：

```text
workspace/runs/<run-id>/transcript/quality.json
```

真实 LLM provider 进入 `adapt` 前必须通过本地 deterministic 质量门禁：

- `safe_to_adapt`：允许进入 DeepSeek / local_vLLM adapt。
- `inspect_first`：阻断，不调用 DeepSeek。
- `reject`：阻断，不调用 DeepSeek。
- 缺少 `quality.json`：阻断，不调用 DeepSeek。

fixture/fake test provider 不受这个门禁影响，因为它们不消耗 DeepSeek 或真实 LLM 资源。

当前默认 DeepSeek 改写风格是 `adapt.style: faithful_spoken`。它要求尽量忠实保留原 transcript 的信息、顺序、语气、问题、数字、人名和观点组织，只做字幕噪声、舞台提示、转写说明、URL/缩写读法等轻清理，并让中文适合 TTS。需要更自然、更松的中文播客化改写时，才在 ignored runtime config 里显式设置 `adapt.style: polished_spoken`。

因此，测试新来源时优先先跑到 `normalize`，看质量报告，再决定是否继续：

```bash
ssh my-5090d-host 'cd /home/th5090d/Develop/personal_project/BabelEcho && .conda/babelecho-dev/bin/python -m babelecho episode convert --workspace workspace --run-id <run-id> --url "<url>" --local-config workspace/config/local-cosyvoice-on-demand-chunked.yaml --to-stage normalize'
```

查看质量报告：

```bash
ssh my-5090d-host 'cd /home/th5090d/Develop/personal_project/BabelEcho && .conda/babelecho-dev/bin/python - <<'"'"'PY'"'"'
import json
from pathlib import Path
run = Path("workspace/runs/<run-id>")
print(json.dumps(json.loads((run / "transcript/quality.json").read_text()), ensure_ascii=False, indent=2))
PY'
```

## 全链路运行

确认 `quality.json` 为 `safe_to_adapt` 后，再跑完整链路：

```bash
ssh my-5090d-host 'cd /home/th5090d/Develop/personal_project/BabelEcho && .conda/babelecho-dev/bin/python -m babelecho episode convert --workspace workspace --run-id <run-id> --url "<url>" --local-config workspace/config/local-cosyvoice-on-demand-chunked.yaml --to-stage publish'
```

如果前面已经跑到 `normalize`，可以从 `adapt` 继续：

```bash
ssh my-5090d-host 'cd /home/th5090d/Develop/personal_project/BabelEcho && .conda/babelecho-dev/bin/python -m babelecho episode convert --workspace workspace --run-id <run-id> --url "<url>" --local-config workspace/config/local-cosyvoice-on-demand-chunked.yaml --from-stage adapt --to-stage publish'
```

如果 DeepSeek chunk 返回 malformed JSON 或临时 429/5xx，当前实现会按 chunk retry 规则重试。若仍失败，先检查 `workspace/runs/<run-id>/run.json`，不要盲目重复全链路。

## Audio-first ASR Smoke

真实 ASR 只在 audio-first 路线验证，不作为 YouTube/RSS/iTunes/article 的 fallback。当前第一步使用 5090D 已安装的本地 `openai-whisper`，不使用云 API。

已确认的 5090D smoke：

- `openai-whisper` 已安装在 `/home/th5090d/miniforge3/envs/babelecho-tts`。
- `torch 2.11.0+cu130` 可用 CUDA，GPU 为 `NVIDIA GeForce RTX 5090 D v2`。
- 公开英文短样本 `jfk.flac` 使用 `tiny.en`、`--device cuda --fp16 True` 可正确产出英文 ASR JSON。
- `audio-asr-jfk-localcli-20260619` 已用 `babelecho audio convert` 跑通真实 `local_cli` ASR：
  - `--to-stage asr` 写出 `asr/raw.json`
  - `provider=openai_whisper`
  - `model=tiny.en`
  - `language=en`
  - `segment_count=2`
  - 从 `--from-stage diarize --to-stage normalize` 继续成功，`quality.recommendation=safe_to_adapt`
- 中文 TTS 样本不要用 `tiny.en --language en` 测质量，否则会产生英文 hallucination；中文样本应使用 multilingual 模型和 `language=zh`。

真实 wrapper 配置示例，放在 ignored runtime config，例如 `workspace/config/local-audio-asr.yaml`：

```yaml
asr:
  provider: local_cli
  command: "/home/th5090d/miniforge3/envs/babelecho-tts/bin/python tools/openai_whisper_asr_wrapper.py"
  model: tiny.en
  language: en
  device: cuda
  extra_args:
    - "--fp16"
    - "true"

diarization:
  provider: none

publish:
  base_url: "https://example.com/babelecho"
```

短音频先放在 ignored 路径，例如 `workspace/sources/audio-asr-smoke.wav`。同步代码后先只跑到 `asr`：


针对上述试听暴露的问题，本轮已实现三层清理：

- `normalize` 会丢弃纯舞台提示段，例如 `[APPLAUSE]`、`[MUSIC]`、`[Radiolab theme music].`。
- `normalize` 会丢弃常见版权尾巴和转写免责声明，例如 `Copyright ... all rights reserved`、`terms of use`、`Transcripts are created ...`。
- DeepSeek adapt prompt 已明确要求清理舞台提示、版权/转写说明，并让 URL、域名、`MP3` / `MP4` 等缩写适合中文 TTS。
- `local_cli` TTS 写入 wrapper `.txt` 前会把 `predictionguard。com`、`99pi.org` 这类域名改成 `predictionguard 点 com`、`99pi 点 org`，并把 `MP三` / `MP3` / `MP四` / `MP4` 改成 `M P 3` / `M P 4`。
- `babelecho check --checks script` 仍会拦截整段英文残留，但允许中文制作名单中夹带英文人名，例如 `Kathy Tu`、`Jayson De Leon`、`Stefan Lawrence`。

仍需人工试听观察：

- 真实节目制作名单、广告口播、赞助说明不默认删除，因为它们可能是节目正文的一部分。
- 带路径的完整 URL、被 LLM 翻成中文数字的域名，以及不同 TTS 模型的缩写读法，后续按真实样本再补更窄规则。
